Swift3.0重学笔记

发表于 2016-11-03   |     |   阅读次数

这里记录了一些零零散散的知识点，有一些可能是3.0之前已经存在，但是没有注意到的知识点，也可能是3.0新增的新特性。

参考文档：The Swift Programming Language

  1. 对于类属性(对类方法同样适用)，使用static修饰。在class类型中，使用class修饰可在子类中覆盖父类的实现。

    ​

  2. 子类在init方法中，必须先初始化子类新增的属性，才能调用父类初始化方法。

  3. extension中不能包含存储属性。

  4. 对于protocol，可以在extension提供默认的实现。

  5. 可以在extension中限定protocol的实现类。

  6. 在protocol中的关联类型，使用关键字associatedtype修饰。

  7. 访问控制中的open和public，open只应用在class以及class成员上，open的访问权限比public高。不同点如下：

    • open修饰的类的派生类可以在其被定义的模块内定义或者其它任何导入了该模块的模块内定义。
    • open修饰的类成员在其派生类中可以在其被定义的模块内覆盖或者其它任何导入了该模块的模块内覆盖。

  8. unit test target下，可以访问internal级别的成员。

  9. 对类成员的get/set方法设置访问限制：interal(set), fileprivate(set), private(set)

  10. swift用五种类型映射C语言中的指针：

C Syntax Swift Syntax
const Type /Type const * UnsafePointer
Type / Type __strong * UnsafeMutablePointer
Type ** AutoreleasingUnsafeMutablePointer
const void * UnsafeRawPoin
void * UnsafeMutableRawPointer

函数指针：@convention(c) ()->Int32 ,在C语言中表示为int(*)(void).

Buffer Pointers:

  • UnsafeBufferPointer
  • UnsafeMutableBufferPointer
  • UnsafeRawBufferPointer
  • UnsafeMutableRawBufferPointer

Null Pointers:

OC Syntax Swift Syntax
const Type * _Nonull UnsafePointer
const Type * _Nullable UnsafePointer?
const Type * _Null_unspecifed UnsafePointer!
  1. C数据类型内存占用计算: MemoryLayout

  1. OC与Swift混合编写(文件导入)

    在OC中导入Swift: 在build setting中的Packaging模块下， Defines Module设为YES，然后在需要用Swift代码的地方#import <ProductName/ProductModuleName-Swift.h>

    在Swift中导入OC: 在项目中建立ProductName-Bridging-Header.h文件，将在Swift需要用到的OC代码在该文件中导入
