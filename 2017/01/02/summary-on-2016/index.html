2016年度总结

发表于 2017-01-02   |     |   阅读次数

生活就像游乐场，尽情去体验吧。

——《无姓之人》

今年还是尝试了挺多的新东西，经历过就知道，对自己而言哪些是好，哪些是不好的。总的来说，工作上、生活上的感悟还是挺多的，一点一点说吧。

工作

  1. 对职业定位的思考。一个人的职业并不是一个定死的东西，它总是会随着行业变化或者是个人意志而有所变化，所以这个东西就需要有一个阶段性的计划了。在整个大方向上，我选择了前端，在当下的短期目标上，我选择了iOS，在更长远的目标上，就需要根据行业发展和个人兴趣做调整了。总说计划赶不上变化，其实不然，一个好的计划，并且按之实行，就可以具备抗灾能力，可以应对意外情况。
  2. 对团队选择的思考。在这方面，经历的并不是很多，但还是有一些想法。团队是由人组成的，所以选择团队其实是在选择哪些人，但是哪些特质的最重要的呢？怎样才算一个良好的技术团队？我自问，我心中的理想的团队是怎样的？人与人之间相处和谐，技术上或是其它方面的东西总是相互分享，生活上热忱相交。所以有两点是我觉得最重要的，一是和谐，二是分享。
  3. 对创业的认知。 以前没有想过创业，更没有经历过创业，所以对这两个字的感觉是很模糊的。什么是创业？为什么创业？怎样创业？怎样才算创业成功？一年的时间还是太短，所得甚少，还是需要保持谦逊之心，且行且看且学之。
  4. 对技术发展的思考。 技术人要想不被淘汰，就必须对技术发展有所关注，这个技术自然也是职业相关的技术。为什么要关注技术发展？一般情况下，同一个问题的更新的解决方案总是更优解，也可能是问题的发展，导致了当下的解决方案不再适用。在自己没有造轮子或者是自己没能力造轮子的情况下，关注领域技术的发展，总是可以让自己以最适当的解决方案去解决问题。

技术

  1. 年初目标完成度。看了今年年初写的计划，有点汗颜，定目标时，心不够静，想法偏多，所以有些尚未完成。
  2. 编程语言。这方面接触偏多些，一开始想学一门脚本语言，于是接触了Python、Ruby，接触后还是有些犹豫，Ruby写起来着实很舒服，但是Python更实用些，最后还是选择深入下Python。JavaScript之前也是有所接触，下半年再次温习了下前端的东西，顺便看了下NodeJS的东西。Swift更别说，2.2与3.0之间的改动还是比较大，上半年看了2.2，下半年就重看了下3.0。
  3. 技术路线。确定了一个比较明确的技术路线—大前端。基本上，以后的技术知识补充都会在这条线上，不会偏到后端去，o(╯□╰)o。
  4. iOS方面。 今年在这方面的深入不足，写了一年的业务代码，当然这是借口。总的来说，还是可以，很多技术细节需要接触，比如基础网络、蓝牙、图片、音视频处理、逆向安全。
  5. 数据结构。 温习了表、树、散列，剩下的来年啃完。
  6. 应用架构。 今年经历了一点点，但还是有些不明白，可能是这些东西在实际实用中的重要性还没凸显出来。还是需要多多看下别人的项目，多听听技术分享，自己也要找机会多经历。
  7. 产品思维。 快到年底才领悟到产品思维，可能跟今年的经历有关，如果技术没办法变成实实在在的产品的话，那学的技术再牛逼也是白搭。如果技术人掌握产品思维，可以将技术转变成产品，那么所掌握的技术才算真正落地。

读书

  1. 为什么读书？如果不能行万里，那一定要读万卷书。读书是认识世界的途径之一，最便宜的那种。

  2. 为什么读哲学？我是功利主义者，因为需要，所以读。我觉得我经常碰到的问题，我无法解决，但似乎哲学上有所说明，所以我觉得有必要去读一读。
